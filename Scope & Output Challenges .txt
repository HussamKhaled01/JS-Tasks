### 1. Hoisting Trap

```
console.log(a)
var a=10
```

🔹 What will be printed? undefined

🔹 Explain why. because a var is hoisted on top but initialized with undefined

### 2. Temporal Dead Zone (TDZ)

```
console.log(a)
let a=10
```

🔹 What happens? caught error reference error 

🔹 Explain TDZ.means that let variables are hoisted to the top of their scope but are not initialized.
From the start of the scope until the declaration line, the variable is in the Temporal Dead Zone, and
 accessing it throws a ReferenceError

### 4. Shadowing + TDZ Trap

```
let x=10

function test() {
console.log(x)
let x=20
}

test()
```

🔹 What happens here? The inner x shadows the global x and JavaScript looks in the function scope first and

The inner x is hoisted but not initialized → TDZ So accessing it throws ReferenceError

```

  }

```

### 15. IIFE Fix (Advanced)

```
for (var i=0;i<3;i++) {
  (function(i) {
setTimeout(() =>console.log(i),0)
  })(i)
}
```

🔹 What is the output? 0 1 2


- `var` → function-scoped (can leak outside blocks)
- `let` → block-scoped (TDZ applies)
- Closures + loops = **VERY common interview trap**
- `setTimeout` → think **event loop + execution context**
- Function declarations are hoisted **before** variables
